# Інформаційна система з обліку туристичного інвентарю з аналітикою використання
Систему розроблено для виконання наступних задач:
## 1. Перегляд інформації про інвентар
- Користувач може переглядати дату входу на склад, назву, ідентифікаційний номер, рівень зносу, кому саме видано, дату видачі та на який проміжок часу видано інвентар.
## 2. Зміна та додавання інформації про інвентар
- Користувач може змінювати будь – який з вищезгаданих параметрів або видаляти інвентар зі списку.
- Користувач може додавати інвентар у список.
## 3. Пошук інвентарю
- Користувач може здійснювати пошук інвентарю за критеріями: ідентифікаційний номер,  назва.
- Користувач може здійснити фільтрацію за станом, доступністю, датою.
## 4. Аналітика використання
- Користувач може переглянути статистику використання інвентарю.
- Користувач може переглянути статистику оренди інвентарю, в тому числі й статистику прострочених повернень.
- Користувач може переглянути статистику зносу інвентарю.

# Як правильно документувати програмний код
## 1. Після заголовку методу, класу, треба створити порожній рядок. <br> Потім в те місце необхідно вставити пару потрійних лапок.
## 2. Перше речення в коментарі має бути результатом кінцевої дії функції/класу та ін. <br> Між документуванням параметрів функції/класу та першим реченням обов`язково має бути порожній рядок!
## 3. Параметри виключення, типи даних та ін., документуються наступним чином:
- Аргументи функції перераховуються окремо через `:param "Назва параметру": "Опис параметру"`. <br> Двокрапки обов'язкові!
- Типи даних аргументів перераховуються окремо під відповідним аргументом через `:type "Назва параметру": "Тип даних параметру"`. <br> Двокрапки обов'язкові!
- Якщо метод щось повертає, це документується з нового рядка через `:return: "Опис того що повертає метод"`. <br> Двокрапки обов'язкові!
- Тип поверненого результату від методу документується з нового рядка окремо під відповідним `:return:`-ом через `:rtype: "Тип поверненого результату"`. <br> Двокрапки обов'язкові!
- Виключення документуються з нового рядка через `:raise: "Опис та тип виключення"` для одного та `:raises: "Описи та типи виключень"` для декількох. <br> Двокрапки обов'язкові!

# Git Hooks
## 1. Для того, щоб створити хук, спочатку необхідно його встановити за допомогою команди `pip install pre-commit`.
## 2. Потім треба виконати команду `pre-commit install`. Після успішного виконання команди, термінал виведе на екран повідомлення, що хук встановлено.
## 3. Після виконання попереднього пункту, необхідно створити та сконфігурувати файл під назвою `.pre-commit-config.yaml`.
## 4. На вкладці `Commit` необхідно зайти в налаштування та вибрати `Git Hooks`.

# Статична типізація
## 1. За допомогою команди `pip install mypy` встановіть бібліотеку для статичної типізації.
## 2. Створіть файл під назвою `mypy.ini` та сконфігуруйте його.
## 3. Перейдіть на вкладку `Settings` -> `Tools` -> `External Tools` та натисніть на значок `+`.
    - Назвіть інструмент як `Mypy`.
    - В поле "Program" введіть `$PyInterpreterDirectory$/mypy`.
    - В поле "Arguments" введіть `$FilePath$ --strict --show-error-codes`.
    - В поле "Working Directory" введіть `$ProjectFileDir$`.
## 4. Збережіть зміни та вийдіть на головний екран IDE.
## 5. Перейдіть на головному екрані на вкладку `Tools` -> `External Tools` -> `Mypy`. <br> При натисканні на програму, вона запуститься (необхідно трохи почекати) та виведе результати роботи в термінал (відкриється автоматично).
